# Fitness Functions: [Название проекта]

> Пример определения fitness functions. Адаптируйте под ваш проект.

## Что это

Fitness functions — автоматизированные проверки архитектурных характеристик, встроенные в CI/CD. Если проверка не проходит — pipeline блокируется.

## Определения

### Performance

| Правило | Порог | Где проверяется | Действие при нарушении |
|---------|-------|----------------|----------------------|
| Латентность API p95 | < 200 мс | CI (нагрузочный тест) | Блокировка деплоя |
| Размер бандла фронтенда | < 500 КБ (gzip) | CI (webpack analyzer) | Warning → блокировка |
| Время старта сервиса | < 30 сек | CI (smoke test) | Блокировка деплоя |

### Reliability

| Правило | Порог | Где проверяется | Действие при нарушении |
|---------|-------|----------------|----------------------|
| Покрытие тестами | > 80% | CI | Блокировка merge |
| Цикломатическая сложность | < 15 на метод | CI (статический анализ) | Warning |
| Нет циклических зависимостей | 0 циклов | CI (dependency check) | Блокировка merge |

### Security

| Правило | Порог | Где проверяется | Действие при нарушении |
|---------|-------|----------------|----------------------|
| Нет критических CVE в зависимостях | 0 critical | CI (сканер зависимостей) | Блокировка деплоя |
| Все API endpoints требуют аутентификации | 100% | CI (API lint) | Блокировка merge |
| Шифрование данных at rest | Все хранилища | IaC policy (OPA) | Блокировка apply |

### Cost

| Правило | Порог | Где проверяется | Действие при нарушении |
|---------|-------|----------------|----------------------|
| Ежемесячные затраты на инфраструктуру | < 500 000 ₽ | Мониторинг (alert) | Уведомление → эскалация |
| Нет инстансов без тегов | 0 без тегов | IaC policy (OPA) | Блокировка apply |

### Architecture

| Правило | Порог | Где проверяется | Действие при нарушении |
|---------|-------|----------------|----------------------|
| API совместимость (breaking changes) | 0 breaking changes | CI (OpenAPI diff) | Блокировка merge |
| Соответствие dependency rules | 100% | CI (ArchUnit/ArchGo) | Блокировка merge |

## Пример CI-шага (GitLab CI)

```yaml
fitness-functions:
  stage: test
  script:
    - npm run test:coverage -- --threshold 80
    - npm run lint:api
    - npm run check:bundle-size
    - npm run check:dependencies
  rules:
    - if: $CI_MERGE_REQUEST_ID
```
