# AI Policy: [Название проекта]

> Пример AI-политики проекта. Адаптируйте под контекст и требования заказчика.

## Разрешённые сценарии использования AI

| Сценарий | Класс инструмента | Ограничения |
|----------|-------------------|-------------|
| Генерация boilerplate-кода | AI-ассистент в IDE | Обязательный code review |
| Рефакторинг и оптимизация | AI-ассистент в IDE, терминальный AI-агент | Обязательный code review |
| Генерация тестов | AI-ассистент в IDE | Проверка покрытия и корректности |
| Анализ требований | Чат-ассистент (LLM) | Результат верифицируется аналитиком |
| Генерация документации | Чат-ассистент (LLM) | Проверка и правка архитектором |
| Code review | Терминальный AI-агент | Дополняет, не заменяет ревью человеком |
| Генерация диаграмм (Mermaid) | Чат-ассистент (LLM) | Проверка корректности |

## Красные зоны (запрещено)

- Отправка production-данных, PII, секретов в AI-инструменты без data processing agreement
- Использование AI для генерации кода безопасности (криптография, аутентификация) без экспертного ревью
- Автоматический деплой AI-сгенерированного кода без code review
- Использование AI-инструментов, не одобренных службой ИБ

## Верификация

| Что проверяем | Как | Кто |
|--------------|-----|-----|
| AI-сгенерированный код | Code review (merge request) | Разработчик + ревьюер |
| AI-сгенерированная документация | Вычитка архитектором | Архитектор |
| AI-сгенерированные тесты | Запуск + проверка покрытия | CI pipeline + разработчик |
| Отсутствие секретов в промптах | Обучение команды, мониторинг | Тимлид |

## Audit Trail

- Все AI-сгенерированные коммиты помечаются в описании (`Co-Authored-By: AI`)
- В ADR указывается, если решение принято с помощью AI-анализа
- Логируется использование AI-инструментов в проектном журнале (опционально)

## Пересмотр

Политика пересматривается каждый квартал или при появлении новых AI-инструментов.
